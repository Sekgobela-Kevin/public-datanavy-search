Datanavy search engine

Welcome to Datanavy search guide. Datanavy search is a custom keywords based search engine for searching files, raw text, and webpages. Visit datanavy.site to access the web application in order to follow the guide. For help regarding Datanavy search contact the developer on email kevin@datanavy.site.

Some older browsers may not be supported including almost all versions of Internet Explorer. It is recommended to use Chrome, MS Edge , Firefox, Safari or Opera with recent version installed.

search result viewed from its resource(Chrome desktop)

Use Datanavy search if:

  • You want to search through uploaded files, etc
  • You want to have control of what to search and other customisations
  • You are a learner or educator
  • You need synonyms of words and other linguistic features
  • You need to analyse structure of sentence
  • You need to detect keywords for your own purposes
  • You are a fan of free stuffs..., try it out here for free

Keywords:

  • Resources – Anything that supplies text data to be searched during a search
  • Query – sentence as like a question that requests for something.
  • Search keywords – Important words within a query which the search engine uses for searching.
  • Relevance – A number between 1- 100(%) representing similarity between search result and query by analysing search keywords.

Basic workflow

Datanavy search on start(Chrome desktop)
  • Step 1: Load resources to be searched(files, raw text, URLs).
  • Step 2: Submit resources and ensure that they are ready for a search.
  • Step 3: Enter search query into search field and press search button
  • Step 4: View search results

Search section

Now with our Wikipedia photosynthesis webpage loaded, it’s time to perform a search to see how Datanavy search works. To visit search section, click search on application navigation bar and the browser screen will move to section responsible for searching.

Search section without search query(Chrome desktop)

To start searching, you would need to enter a search query. The search query should be rich of relevant keywords with no spelling errors. Remember that Datanavy search is keywords based and under development.

Search with query entered(Chrome desktop)

Please note that only keywords in the entered sentence(query) are part of search. The rest of the words have no significance in the search.

Keywords from above query may be process, photosynthesis and plants. They were detected using machine learning and artificial intelligence combined with knowledge of English.

Search query analysis(Chrome desktop)

Image above displays results of search query analysis(sentence analysis). Bolded words in yellow background represent keywords. Parts of speech, synonyms and other English language features have been detected.

Sometimes it may be slow to get search query analysis due to long complex computations. The analysis may be used for wide variety of tasks other than optimizing search query.

Search ready resources information(Chrome desktop)

Image above displays search ready resources directly in resource section. Any resource listed is part of a search, if you want to include or exclude a resource then you should do it in resources section.

Name, type, length in words, length in characters and whether the resource is processed are displayed. The listed data may determine how long the search may take, with larger resources like files taking long to be searched.

Processed resources are resources that are already processed and saved. Processed resources get searched faster than not raw resources due to their optimised nature. Resources get processed the first time they are searched which causes first search to be slow compared to next searches.

Search settings(Chrome desktop)

    Search settings

  • Minimum relevance - Excludes results with relevance less than the value :: default: 50%
  • Maximum results - Sets maximum limit on number of results search should return :: default: 100000
  • Results per view - Set maximum results that can be displayed in a view :: default: 4
  • Search all resources - Searches all resources including disabled ones :: default: inactive
  • First results only - Discards all results that cannot fit into the view leaving default: inactive

One last thing before searching is checking search settings. Settings allows you to filter the search results instead of search engine deciding for you. The settings can also be changed after searching yet filtering the search results.

Settings at top influences the future searches results while bottom settings only apply to current search results. The same applies to search query analysis and search resources.

search results after clicking search button(Chrome desktop)

Now click search button and see magic. After searching 4 search results appear with the rest excluded. Remember Results per view and Maximum resultsin settings, you can change those in search settings according to your expectations.

Note the white area at bottom of search result. It functions as to display the resource in which the result was extracted from. Our result was extracted from Wikipedia photosynthesis webpage which is the webpage uploaded in resource section.

Bolded words represent keywords from search query and yellow part represent text surrounding the keywords. Brown part represent section on which the search result was extracted.

Features

Note that the application is using custom search algorithm which is under development. The application both at server and client is under development with new feature tested on daily basis.

Every feature in the application took lot of efforts to build and incorporate with existing features. New feature will be added to help improve search engine precision and user experience. The Guide may also be out of date due to more features and removed in short period.

The search engine support extraction of data from pdf, pptx, docx and text(not limited to *.txt) files. Extracting files is a slow, computer intensive operation especially pdf files. In rare cases, a file may not be extracted.

Also supports webpages as resource giving ability to use rich internet data for searching. The search engine uses the supplied URLs to locate and extract webpages from their location in the World Wide Web.

Traditional typing is also supported for those with text data in their fingertips or clipboard. It’s better to use clipboard than manually typing(impractical). This is the fastest method of loading a resource compared to webpages and files.

For searching, it is build-in with Google search. The Google search within is called Google search console built by Google to offer web applications access to Google search. Its purpose is to give access to Google search and provide webpage URLs to be used as resources.

Supports sentence analysis with search query analysis. This feature extract linguistic features of sentence(query) such as synonyms, parts of speech, entity recognition, keywords detection , stop word detection and much more. Search query analysis was strictly intended for keywords identification and search query optimization to improve search results relevance.

Application also offers powerful highlighting, both the keywords and surrounding text are highlighted using different colours. This gives an opportunity to understand which keywords were matched and how they relate to each other within the matched sentence/paragraph.

Another powerful feature is ability to view the result within its original resource. It makes it easier to locate which part of resource text the result got retrieved. Relevance for each search result is displayed on top of the search results.

Search results can be customized such as setting minimum relevance and other settings affecting display of search results. It all depends on how strict you expect the search to be, setting high minimum relevance means that more result would be filtered out leaving less results and setting minimum relevance of 1% would include wide variety of results.

Datanavy search is also very fast. It was built with speed and performance in mind especially at server side were computations are performed. With more hardware, it will become much faster and support large number of resources due to increased storage.


Compared to web search engines

The search engine does not in any way compete with advanced search engines(Google, Bing, Yahoo, etc). But it offers an opportunity to have control of what to be searched. The application contains built-in Google to help offer features of advanced search engines.

Google search was designed to use advanced technologies for searching the whole web in short time with much higher relevance. Datanavy was designed for using keywords to locate which parts of loaded text query matched.

Web search engines search billions of documents and support wide variety of sources including images, videos, audios and much more. For something very powerful, web search engines as Google are necessary. They can search billions of webpages in less than a second using powerful servers and algorithms. They are also based on context not on keywords giving them an opportunity find the best match to query. Datanavy search is using custom search algorithm which is currently keywords based but it’s still under development.

Because the web is very rich with data, that is why webpages are supported as resource/source. You will use your own search engine and then Copy (Ctrl + C) and Paste (Ctrl +V) hyperlink into the application. That way you will be combining the power of web search engines with Datanavy search.

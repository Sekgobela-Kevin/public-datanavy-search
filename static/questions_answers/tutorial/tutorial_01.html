DATANAVY SEARCH ENGINE

Datanavy search is a custom, local keywords-based search engine for searching files, raw text, and webpages. Local means it searches resources loaded at user side as opposed to those available at the server or world wide web.

Keywords:

  • Query – sentence as like a question that requests for something.
  • Search keywords – Important words within a query which the search engine uses for searching.
  • Local – Resources in which you are the only on with access to them.
  • Resources – Anything that supplies text data to be searched during a search
  • Relevance – A number between 1- 100(%) representing similarity between search result and query by analysing search keywords.

BASIC WORKFLOW

Datanavy search on start(Chrome desktop)
  • Step 1: Load resources to be searched(files, raw text, URLs).
  • Step 2: Submit resources and ensure that they are ready for a search.
  • Step 3: Enter search query into search field
  • Step 4: View search results

SEARCH

Now with our Wikipedia photosynthesis webpage loaded, it’s time to perform a search to see how datanavy search works. To visit search section, click search on application navigation bar and the browser screen will move to section responsible for searching.

Search section without search query(Chrome desktop)

To start searching, you would need to enter a search query. The search query should be rich of relevant keywords with no spelling errors. Remember that datanavy search is keywords based and under development.

Search with query entered(Chrome desktop)

Please note that only keywords in the entered sentence(query) are part of search. The rest of the words have no significance in the search.

Keywords from above query may be process, photosynthesis and plants. All of which are detected using machine learning and artificial intelligence combined with knowledge of English.

Search query analysis(Chrome desktop)

Image above displays results of search query analysis. Bolded words in yellow background represent keywords. Parts of speech, synonyms and other English language features.

Sometimes it may be slow to get search query analysis due to complex computations. The analysis may be used for wide variety of things other than optimizing search query.

Search ready resources information(Chrome desktop)

Image above displays search ready resources directly into resource section. Any resource listed is part of a search, if you want to include or exclude a resource then you should do it in resources section.

Name, type, length in words, length in characters and whether the resource is processed are displayed. The listed data may determine how long the search may take with larger resources like files taking long to be searched.

Processed resources are resources that are already processed and saved. Processed resources get searched faster than not raw resources due to their optimised nature. Resources get processed the first time they are searched which means first search is slow compared to following searches.

Search settings(Chrome desktop)
  • Minimum relevance - Excludes results with relevance less than the value :: default: 50%
  • Maximum results - Sets maximum limit on number of results search should return :: default: 100000
  • Results per view - Set maximum results that can be displayed in a view :: default: 4
  • Search all resources - Searches all resources including disabled ones :: default: inactive
  • First results only - Discards all results that cannot fit into the view leaving default: inactive

One last thing before searching is checking search settings. Settings allows you to filter the search results instead of search engine deciding for you. The settings can also be changed after searching yet filtering the search results.

search results after clicking search button(Chrome desktop)

Now click search button and see magic. After searching 4 search results appear with the rest excluded. Remember Results per view in settings, you can change search results view using the numbers at top of results.

Number in red circle represent search relevance displayed search result having relevance of 99%. Relevance is higher as all keywords have been matched. Relevance is too dependent on keywords.

Note the white area at bottom of search result. It functions as to display the resource in which the result was extracted from. Our result was extracted from Wikipedia photosynthesis webpage which is the webpage uploaded in resource section.

Bolded words represent keywords from search query and yellow part represent text surrounding the keywords. Brown part represent section on which result was extracted from.

search result viewed from its resource(Chrome desktop)

Click expand icon at bottom right of result to view the search full within its resource. Currently the resource is in text form performance purposes. Now you can easily spot which part of resource the search result was extracted from.

You could also copy the search result to clipboard by clicking copy icon on bottom left of search result.

FEATURES

Note that the application is using custom search algorithm which is under development. The application both at server and client is under development with new feature tested on daily basis.

Every feature in the application took lot of efforts to build and incorporate with other features. New feature will be added with datanavy not limited to search engine developed further.

The search engine support extracting of data from pdf, pptx, docx and text(not limited to *.TXT) files. Files are very slow to be extracted especially larger pdf. In rare cases, a file may not be extracted but it that has never been reported or spotted.

Also supports webpages as resource giving ability to use rich internet data for searching. The search engine uses the supplied URL to locate and extract webpage from its location in the world wide web.

Traditional typing is also supported for those with text data in their fingertips or clipboard. It’s better to use clipboard than manually typing(impractical). This is the fastest method of loading a resource compared to webpages and files.

For searching, it is build-in with Google. The Google search within is called Google search console built by Google ink to offer web applications access to Google. Its purpose is to give access to Google search and provide webpage URLs to be used as resources.

Supports sentence analysis with search query analysis. This feature extract linguistic features of sentence(query) such as synonyms, parts of speech(precise),categories identification, keywords detection, stop word detection, entity recognition and much more. Search query analysis was strictly intended for keywords identification and search query optimization to improve search results relevance.

Application offers powerful is highlighting, both the keywords and the text within search result are highlighted using different colours. This gives an opportunity to know which keywords were matched and how they relate to each other within the matched sentence/paragraph.

Another powerful feature is ability to view the result within its original resource. It makes it easier to locate which part of resource text the result came from. Relevance for each search result is displayed on top of the search results.

Search results can be customized such as setting minimum relevance and other settings affecting display of search results. It all depends on how strict you expect the search to be, setting high minimum relevance means that more result would be filtered out leaving less results and setting minimum relevance of 1% would include wide variety of results.

COMPARING TO WEB SEARCH ENGINES

The search engine does not in any way compete with advanced search engines(Google, Bing, etc.). But it offers an opportunity to have control of what to be searched. The application contains built-in Google to help offer features of advanced search engines. Google was designed to use advanced technologies for searching the whole web in short time with much higher relevance. Datanavy was designed for using keywords to try to locate which parts of loaded text query matched.

Web search engines search millions of documents and support wide variety of sources including images, videos, audios and much more. For something very powerful, web search engines as Google are necessary for those tasks. They can search billions of webpages in less than a second using powerful servers and algorithms. They are also based on context not on keywords giving them an opportunity find the best match to query. Datanavy search is using custom search algorithm which is currently keywords based but it’s still under development.

Because the is very rich with data, that is why webpages are supported as resource/source. You will use your own search engine and then Copy (Ctrl + C) and Paste (Ctrl +V) hyperlink into the application. That way you will be combining the power of web search engines with Datanavy search.
